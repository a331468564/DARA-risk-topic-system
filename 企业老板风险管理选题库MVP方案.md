# 企业老板风险管理选题库 MVP 方案

## 1. 项目目标

搭建一个低成本、开源优先的「企业相关风险 / 企业老板风险管理」选题库系统。

核心目标不是一开始做完整风控系统，而是先跑通：

**信息源采集 → 风险关键词筛选 → 飞书多维表入库 → AI/Agent 生成选题 → 人工核验 → 输出内容选题**

第一阶段目标：

- 每天沉淀 5-20 条企业风险素材；
- 每天生成 5-10 个企业老板视角的文案选题；
- 所有素材统一进入飞书多维表管理；
- 优先使用免费、开源、可本地部署工具。

## 2. 数据源范围

当前计划观察以下数据源：

- 华尔街见闻
- 财新网
- 彭博社
- 雪球
- 财联社
- 36氪
- 企查查

处理原则：

- 能免费 RSS / 网页订阅的优先接入；
- 不能免费稳定接入的，先跳过自动化；
- 付费源只做人工参考，不进入第一版自动化；
- 企查查第一阶段不接 API，只做人工核验；
- 禁止优先考虑爬虫强抓，避免稳定性和合规问题。

## 3. 数据源分层

### 3.1 新闻信息源

用于发现企业、行业、市场、政策、资本相关风险信号。

候选来源：

| 来源 | 侧重领域 |
|------|----------|
| 华尔街见闻 | 宏观、市场、行业风险 |
| 财新网 | 监管、商业调查、深度风险背景 |
| 彭博社 | 全球市场、跨国企业、资本市场风险 |
| 雪球 | 投资者情绪、个股讨论、舆情线索 |
| 财联社 | A股快讯、上市公司事件、监管动态 |
| 36氪 | 创业公司、融资、科技公司、创投风险 |

第一阶段处理方式：

1. 优先寻找官方 RSS；
2. 没有官方 RSS 的，尝试 RSSHub；
3. RSSHub 不稳定的，暂时人工查看；
4. 需要登录、会员、付费的源，不做自动化抓取；
5. 只保存标题、摘要、链接、来源、时间，不保存全文。

### 3.2 企业风险核验源

用于核验企业是否存在真实风险信号。

候选来源：

- 企查查
- 国家企业信用信息公示系统
- 中国执行信息公开网
- 信用中国
- 巨潮资讯网
- 交易所公告
- 公司官网公告

第一阶段处理方式：

- 企查查不接 API；
- 企查查仅用于人工核验；
- 能公开查询的官方平台优先作为证据来源；
- 涉及具体企业和老板时，必须标记"已核验 / 未核验"；
- 未核验内容不能直接作为确定性结论输出。

## 6. CLI 接入思路

CLI 的目标是把素材快速写入飞书多维表。

### 6.1 CLI 名称

暂定：`risk`

### 6.2 第一版命令

只做 3 个命令：

- `risk add`
- `risk import`
- `risk report`

### 6.3 命令说明

**添加单条素材**

```bash
risk add \
  --source "财联社" \
  --title "某企业新增被执行信息" \
  --url "https://example.com/news/1" \
  --company "某某公司" \
  --risk-type "债务风险" \
  --risk-signal "被执行"
```

作用：将单条风险素材写入飞书多维表。

**批量导入素材**

```bash
risk import ./data/rss_today.json
```

作用：从 RSS / JSON / CSV 文件批量导入飞书多维表。

**生成今日报告**

```bash
risk report --today
```

作用：读取今天新增的风险素材，按风险类型分组，生成待核验清单和可写选题清单。

## 7. MCP 接入思路

MCP 的目标是让 Agent 能够读写飞书多维表。

优先接入：飞书 / Lark 官方 OpenAPI MCP

- 只启用多维表相关能力
- 不要一开始开放全部飞书权限

建议只开启：

- 多维表 Base 查询记录
- 多维表 Base 新增记录
- 多维表 Base 更新记录
- 多维表 Base 字段查询

MCP 使用场景：

1. Agent 查询今天新增的风险素材；
2. Agent 查询"未核验"的素材；
3. Agent 查询某家公司历史素材；
4. Agent 根据素材生成老板风险选题；
5. Agent 更新核验状态；
6. Agent 把生成的选题写回多维表。

## 8. 最小技术架构

第一版架构：

```
RSSHub / FreshRSS / 手动素材
        ↓
risk CLI / n8n / 手动导入
        ↓
飞书多维表：企业老板风险选题库
        ↓
飞书 MCP
        ↓
Agent 查询、整理、生成选题、更新状态
```

开源优先组件：

| 组件 | 用途 |
|------|------|
| RSSHub | 用于生成 RSS 源 |
| FreshRSS | 自建 RSS 阅读器 |
| changedetection.io | 网页变化监控 |
| n8n | 自动化工作流 |
| lark-mcp | 飞书 MCP 接入 |
| 自写 risk CLI | 批量写入飞书多维表 |

## 9. 第一阶段不做的事情

为了控制成本和复杂度，第一阶段不做：

- 不接企查查付费 API；
- 不做大规模爬虫；
- 不抓取付费全文；
- 不做完整企业风控评分；
- 不做老板个人画像；
- 不做自动发布；
- 不做复杂数据库；
- 不做商业化后台。

## 10. 第一阶段执行计划

### Day 1：建飞书多维表

**任务：**

- 创建「企业老板风险选题库」
- 创建主表「风险素材表」
- 配置基础字段
- 手动录入 5 条样例数据

**验收标准：**

- Agent 能看懂表结构；
- 人工能正常录入素材；
- 字段足够支持后续选题生成。

### Day 2：接入 MCP

**任务：**

- 创建飞书自建应用
- 获取 APP_ID 和 APP_SECRET
- 申请多维表权限
- 把应用加入多维表协作者
- 配置 lark-mcp
- 只开启多维表工具

**验收标准：**

- Agent 能读取多维表记录；
- Agent 能新增一条测试记录；
- Agent 能更新一条测试记录。

### Day 3：写最小 CLI

**任务：**

- 实现 `risk add`
- 支持命令行传入 source/title/url/company/risk_type/risk_signal
- 写入飞书多维表

**验收标准：**

- 执行 `risk add` 后，飞书多维表出现新记录。

### Day 4：接入 RSS / 手动源

**任务：**

- 尝试 RSSHub / FreshRSS
- 优先接 36氪、财联社、华尔街见闻等可接入源
- 无法免费接入的源先跳过
- 输出 rss_today.json

**验收标准：**

- 每天能拿到 20 条以内素材；
- 素材包含标题、链接、来源、时间。

### Day 5：关键词过滤

**任务：**

- 实现关键词命中逻辑
- 命中关键词的素材进入飞书
- 未命中的素材先丢弃或进入候选池

**验收标准：**

- 系统能筛出"被执行、限高、处罚、股权冻结、问询函"等相关素材。

### Day 6：Agent 生成选题

**任务：**

- Agent 查询当天新增素材
- 按风险类型分组
- 生成老板视角选题
- 写回飞书字段

**验收标准：**

- 每条有效素材至少生成 3 个可写选题。

### Day 7：复盘

**任务：**

- 检查哪些来源有效
- 检查哪些关键词误报高
- 检查哪些风险类型最适合内容创作
- 决定是否继续自动化

**验收标准：**

- 形成第一版稳定流程；
- 每天可以产出 5 条以上可用选题。

## 13. Agent 可用提示词

### 13.1 生成选题

```
你是企业老板风险管理选题助手。

请从飞书多维表「企业老板风险选题库」中读取今天新增、处理状态为"待加工"的素材。

请完成：
1. 判断风险类型；
2. 提炼风险信号；
3. 转换成老板视角；
4. 为每条素材生成 3 个文案选题；
5. 为每个选题生成一个短视频开头；
6. 如果证据不足，标记为"需进一步核验"；
7. 将结果写回飞书多维表。
```

### 13.2 核验风险

```
你是企业风险素材核验助手。

请检查飞书多维表中核验状态为"未核验"的记录。

对每条记录判断：
1. 是否涉及具体企业；
2. 是否涉及具体老板、法人、实控人；
3. 是否有公开证据链接；
4. 是否适合作为内容案例；
5. 是否存在事实风险或名誉风险。

请输出：
- 可用
- 需进一步核验
- 不建议使用
- 原因
```

### 13.3 生成日报

```
你是企业老板风险选题日报助手。

请读取飞书多维表中今天新增的风险素材。

按以下格式输出：
1. 今日风险素材总数；
2. 各风险类型数量；
3. 最值得写的 5 条；
4. 每条对应 3 个老板视角选题；
5. 需要人工核验的素材；
6. 明天建议重点关注的关键词。
```

## 14. MVP 判断标准

第一版只要满足以下标准，就算成功：

1. 能把风险素材写入飞书多维表；
2. 能按关键词筛选出企业风险相关素材；
3. Agent 能读取飞书多维表；
4. Agent 能生成老板视角选题；
5. Agent 能把结果写回飞书；
6. 每天能沉淀至少 5 条可用选题；
7. 全流程不依赖付费 API。

## 15. 当前优先级

**最高优先级：**

- 飞书多维表字段设计
- 飞书 MCP 接入
- risk add CLI
- 关键词过滤
- 今日选题生成

**低优先级：**

- 企查查 API
- 复杂爬虫
- 自动风控评分
- 自动发布
- 多用户权限
- 商业化后台

## 16. 总结

第一版不要做"大而全的企业风控系统"。

应该先做：**开源低成本的企业老板风险选题库 MVP**

核心闭环是：

```
免费信息源 → 关键词筛选 → 飞书多维表 → Agent 生成选题 → 人工核验 → 内容输出
```

如果 MVP 跑通，再逐步升级：

- **阶段 2**：n8n 自动化
- **阶段 3**：接入更多公开数据源
- **阶段 4**：企查查 API / 付费数据源
- **阶段 5**：企业风险日报
- **阶段 6**：行业风险专题库
